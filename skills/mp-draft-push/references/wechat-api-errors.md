# 微信公众号文章 API 错误码速查

> 基于 2026-05-24 实测。微信 API 可能更新，需要重新验证。

## 草稿箱 API 错误码

| 错误码 | 含义 | JSON 字段名 | 常见原因 | 解决方案 |
|--------|------|-------------|---------|---------|
| 45003 | title size out of limit | `title` | 标题超 64 字节，或含敏感字符 | 用 Python 检查字节数 `len(s.encode('utf-8'))`，控制在 60 字节以内，避免全角括号/冒号 |
| 45004 | description size out of limit | `digest` | 摘要超长，或含非法字符 | 直接设 `digest: ""` 让微信自动生成，不要手动写摘要 |
| 45110 | author size out of limit | `author` | 作者名含空格/全角字符 | 用简单的名字：2-4 个汉字，避免空格和标点 |

## 经验教训（2026-05-24 实测）

### 标题 · 45003
- `从零开始Hermes（二）：跑起来吧，一行命令的事` = 25 字符 / 63 字节 → ❌ 拒绝
- `从零开始Hermes(2)跑起来吧` = 12 字符 / 33 字节 → ✅ 通过
- **结论**：即使字节数低于 64，含全角符号的标题也可能被拒。安全上限定在 60 字节以下。

### 摘要 · 45004
- `手把手部署Hermes Agent，从系统要求到第一个对话，踩坑经验全分享。` = 38 字符 → ❌ 拒绝
- `""` (空字符串) → ✅ 通过
- **结论**：微信 API 的 `description` (digest) 字段实际限制比文档更严格。留空最安全。

### 作者 · 45110
- `夜航的 Agent 笔记` = 22 字节 → ❌ 拒绝
- `夜航` = 6 字节 → ✅ 通过
- **结论**：避免作者名中出现空格（" "）或全角字符。

## 通用建议

任何字段被拒时：
1. 先尝试设空字符串或最短的可用值
2. 用 Python 计算字节长度 (`len(s.encode('utf-8'))`)
3. 避免全角符号、空格、特殊字符
4. 错误信息中的字段名（title/description/author）可能和 API 文档中的字段名不一致
